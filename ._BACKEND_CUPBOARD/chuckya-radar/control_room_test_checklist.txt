──────────────────────────────────────────
 1. INFRASTRUCTURE
──────────────────────────────────────────
[ ] Redis running and reachable
    → docker compose up redis
    → redis-cli ping → PONG

[ ] Docker Compose stack healthy
    → cd chuckya-radar && docker compose up -d
    → docker compose ps (all 6 services UP)

[ ] Ports open and not conflicting:
    8081 → radar-api
    8082 → static dashboard
    8083 → device-verifier
    8084 → ws-server
    9091 → metrics-server
    6379 → redis

──────────────────────────────────────────
 2. BACKEND API SMOKE TESTS
──────────────────────────────────────────
[ ] Health check
    curl http://localhost:8081/healthz
    → {"status":"ok","service":"chuckya-radar"}

[ ] Unsigned ingest
    curl -X POST http://localhost:8081/v1/radar/event \
      -H "Content-Type: application/json" \
      -d '{"type":"manual","riskScore":75,"source":"test"}'
    → {"id":"R-...","status":"ingested",...}

[ ] List alerts
    curl http://localhost:8081/v1/radar/alerts
    → JSON array with at least 1 alert

[ ] Device public key registration
    curl -X POST http://localhost:8083/v1/device/registerPublicKey \
      -H "Content-Type: application/json" \
      -d '{"appInstanceId":"test-dev-1","publicKeyBase64":"<base64-pem>"}'
    → {"status":"registered",...}

[ ] Consent storage
    curl -X POST http://localhost:8083/v1/device/consent \
      -H "Content-Type: application/json" \
      -d '{"appInstanceId":"test-dev-1","consent":{"appId":true,"imei":false,"location":true}}'
    → {"status":"consent_stored",...}

[ ] Metrics endpoint
    curl http://localhost:8081/metrics
    → Prometheus text format with radar_alerts_total, radar_ingest_latency_seconds, etc.

──────────────────────────────────────────
 3. WEBSOCKET SERVER
──────────────────────────────────────────
[ ] WS health check
    curl http://localhost:8084/healthz
    → {"status":"ok","clients":0}

[ ] WS connect + auth (use wscat or similar)
    wscat -c ws://localhost:8084
    → Send: {"type":"auth","token":"<jwt-with-regions>"}
    → Receive: {"type":"auth_ok",...}
    → Send: {"type":"subscribe","region":"brisbane"}
    → Receive: {"type":"subscribed","region":"brisbane"}

[ ] Admin publish → WS delivery
    (Keep wscat open, in another terminal:)
    curl -X POST http://localhost:8084/publish \
      -H "Authorization: Bearer admintoken" \
      -H "Content-Type: application/json" \
      -d '{"region":"brisbane","payload":{"alertId":"test-1","mode":"code_black","riskScore":95}}'
    → wscat receives: {"type":"alert","region":"brisbane","payload":{...}}

[ ] Latency < 1 second (visual check in wscat timestamps)

──────────────────────────────────────────
 4. DASHBOARDS
──────────────────────────────────────────
[ ] HTML control room
    Open http://localhost:8082/control_room.html
    → 6 panels visible, neon theme, radar sweep animating

[ ] Simple dashboard
    Open http://localhost:8082/
    → Alert list populates after ingest

[ ] React control room (if built)
    cd chuckya-radar/control-room-frontend
    npm install && npm run dev
    → Opens on http://localhost:3000
    → 6-panel grid with live data from API

──────────────────────────────────────────
 5. EVIDENCE & POLICE EXPORT
──────────────────────────────────────────
[ ] Export evidence ZIP
    curl -X POST http://localhost:8081/v1/radar/alerts/<alertId>/export \
      --output evidence.zip
    → ZIP contains alert.json, manifest.json, chain_of_custody.json

[ ] Verify SHA256 integrity
    unzip evidence.zip -d evidence_check/
    sha256sum evidence_check/alert.json
    → Matches hash in manifest.json
