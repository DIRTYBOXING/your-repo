# DataFightCentral Data Models

This document outlines the 14 data models used in the DataFightCentral platform.

## Core Models

- **User:** Represents a user of the app.
- **Fighter:** Represents a combat sports fighter.
- **Gym:** Represents a gym.
- **Event:** Represents a fight event.
- **Fight:** Represents a single fight within an event.
- **Ranking:** Represents a fighter's ranking.
- **FighterBranding:** Represents a fighter's brand.

## Community Models

- **Post:** Represents a social media post.
- **Comment:** Represents a comment on a post.
- **Job:** Represents a job listing.
- **News:** Represents a news article.

## Analytics Models

- **FightStock:** An engagement index for fighters.
- **EngagementMetrics:** Tracks likes, comments, and shares.
- **UserActivity:** Logs user actions.
- **AdPerformance:** Tracks ad impressions and clicks.
- **AIRecommendation:** Stores AI-generated recommendations.

## Privacy Models

- **Consent:** Manages user consent for data collection.
- **Verification:** Tracks user verification status.
- **AuditLog:** Logs important actions for security.

## Health Models

- **HealthLog:** Stores user-provided health data.
- **AIInsight:** Stores AI-generated health insights.
