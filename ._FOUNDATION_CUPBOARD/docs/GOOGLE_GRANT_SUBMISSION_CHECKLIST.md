# Google Grant Submission Checklist

Use this file as your command center for Google for Nonprofits, Ad Grants, and Cloud credits submission.

## 1. Legal Readiness

- [ ] Legal nonprofit name exactly matches registration documents
- [ ] EIN available and verified
- [ ] Nonprofit status approved (or local equivalent accepted by validation partner)
- [ ] TechSoup/Percent validation completed
- [ ] Primary authorized signatory identified

## 2. Website Readiness

- [ ] Domain resolves correctly over HTTPS
- [ ] Homepage clearly states mission and public benefit
- [ ] Programs page explains DataFightCentral and impact model
- [ ] Contact page includes valid email and address details
- [ ] Privacy Policy page is public
- [ ] Terms of Service page is public
- [ ] Donation or support pages contain no policy violations
- [ ] Site is mobile responsive and loads reliably

## 3. Application Pack Readiness

- [ ] `GOOGLE_GRANT_APPLICATION.md` has no unresolved placeholders
- [ ] Founder/CEO contact details are final
- [ ] Signature block and date are complete
- [ ] Demo URL is live and publicly accessible
- [ ] Video walkthrough URL is live and publicly accessible
- [ ] Appendix links open correctly

## 4. Google Ad Grants Readiness

- [ ] Campaign structure documented (`docs/GOOGLE_GRANTS_STRATEGY.md`)
- [ ] Keyword plan documented (`docs/google_grants_campaigns.csv`)
- [ ] Conversion tracking plan defined
- [ ] Negative keyword list prepared
- [ ] Landing pages mapped to each ad group
- [ ] Governance rule: weekly optimization owner assigned

## 5. Cloud Credits Readiness

- [ ] 12-month cost estimate reviewed and updated
- [ ] Credit ask amount clearly stated
- [ ] Services and usage assumptions justified
- [ ] Impact metrics linked to infrastructure usage

## 6. Evidence Bundle (Attach or Link)

- [ ] GitHub repository URL
- [ ] Product screenshots (core flows)
- [ ] Architecture document (`docs/ARCHITECTURE.md`)
- [ ] Community guidelines (`docs/COMMUNITY_GUIDELINES.md`)
- [ ] Data model and schema docs
- [ ] Deployment and implementation docs

## 7. Day-of-Submission QA

- [ ] Every URL clicked and verified
- [ ] Contact email tested by sending and receiving a message
- [ ] Organization name, EIN, and address consistency check completed
- [ ] Mission statement consistency check completed across all forms
- [ ] Grammar and clarity pass completed

## 8. Post-Submission Follow-Through

- [ ] Confirmation emails archived
- [ ] Application IDs logged
- [ ] Follow-up calendar reminders set (7, 14, and 28 days)
- [ ] Impact dashboard baseline captured before campaigns start

## Quick Copy Blocks

Mission one-liner:
`DataFightCentral is a safety-first mental health and community platform for combat athletes, designed to prevent crisis, reduce isolation, and connect users to verified support before harm occurs.`

Primary ask:
`We are requesting Google for Nonprofits activation, Google Ad Grants eligibility ($10,000/month), and Google Cloud credits ($10,000/12 months) to scale crisis prevention and verified support pathways for combat athletes.`
