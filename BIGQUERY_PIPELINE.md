# 🚀 DFC BIGQUERY PIPELINE

**STATUS:** READY FOR DEPLOYMENT

To move Data Fight Central into an enterprise-grade analytics era, we stream our critical Firestore collections (`telemetry`, `revenueEvents`, `ppvPurchases`, `fightStats`) directly into Google BigQuery in real-time.

This allows Vertex AI to train custom fight prediction models on massive datasets, and gives Promoters the ability to generate complex financial reports without overloading Firestore reads.

## 1. Installation (Firebase CLI)
Run this command in your terminal. This installs the official "Stream Firestore to BigQuery" extension.

```bash
firebase ext:install firebase/firestore-bigquery-export --project=datafightcentral
```

## 2. Configuration Parameters
During installation, the CLI will prompt you for the following parameters. You will need to install this extension **once for each collection** you want to track.

**Example: Exporting the Telemetry Collection**
- **Cloud Functions location:** `us-central1`
- **BigQuery Dataset ID:** `dfc_analytics`
- **Collection path:** `telemetry`
- **Table ID:** `raw_telemetry`

**Example: Exporting the Revenue Collection**
- **Collection path:** `revenueEvents`
- **Table ID:** `raw_revenue`

## 3. How It Works
Once installed, Firebase automatically creates a hidden Cloud Function. 
Every time a document is created, updated, or deleted in your targeted Firestore collections, that function intercepts the change and streams the raw JSON into BigQuery.

## 4. Example: Promoter Revenue View in BigQuery
Once your data is flowing into BigQuery, you can use standard SQL to generate massive reports instantly. 

For example, to find out the total revenue generated by a specific promoter this month, you would run this in the BigQuery SQL Workspace:

```sql
SELECT 
  JSON_EXTRACT_SCALAR(data, '$.participants.promoter') AS promoter_id,
  SUM(CAST(JSON_EXTRACT_SCALAR(data, '$.amountCents') AS INT64)) / 100 AS total_revenue_dollars,
  COUNT(*) AS total_transactions
FROM 
  `datafightcentral.dfc_analytics.raw_revenue_raw_changelog`
WHERE 
  JSON_EXTRACT_SCALAR(data, '$.revenueType') = 'ppv'
GROUP BY 
  promoter_id
ORDER BY 
  total_revenue_dollars DESC;
```

## 5. Vertex AI Integration
By having your `telemetry` and `fights` data inside BigQuery, you can now use **Vertex AI AutoML** to point directly at your `dfc_analytics` dataset and say:

*"Train a model to predict the winner of a fight based on these telemetry trends."*

It requires zero data exporting or Python scripting on your end. The pipeline is fully managed.
