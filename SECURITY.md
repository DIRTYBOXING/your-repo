# Security Policy

## Reporting a vulnerability
Report suspected security issues privately to the maintainers before public disclosure.

- Contact: add a dedicated security email address or monitored inbox
- Include: affected area, reproduction steps, impact, and any logs or screenshots that help validate the issue

## Response targets
- Initial acknowledgement: within 5 business days
- Triage decision: within 10 business days when reproducible details are provided
- Remediation timeline: based on severity and deploy risk

## Scope
This repository contains active deployment workflows and service code. Do not publicly post secrets, tokens, or exploit details before maintainers confirm remediation.

## Safe harbor
Good-faith research intended to improve security will be reviewed constructively, provided it avoids privacy violations, service disruption, or data destruction.
