# DFC Mux Owner Request

Status: send this to the DFC-side owner of the Mux account, not to promoters by default.

Purpose: get the exact five values required to switch DFC from rehearsal mode to real Mux-backed live PPV.

## Who Should Own This

- DFC should own the Mux account and all five secrets.
- Promoters should receive stream credentials generated by DFC, not raw Mux API credentials.
- Only use promoter-owned Mux credentials when there is a deliberate bring-your-own-broadcast arrangement.

## Exact Message To Send

Use this message as-is:

```text
Need the five Mux production values for Data Fight Central so we can move the PPV lane from rehearsal mode to real live stream mode.

Please send:
1. MUX_TOKEN_ID
2. MUX_TOKEN_SECRET
3. MUX_SIGNING_KEY_ID
4. MUX_SIGNING_PRIVATE_KEY
5. MUX_WEBHOOK_SECRET

These are for the DFC-owned Mux account. We are not asking promoters for raw Mux API keys.

Once received, we will load them into Firebase Functions secrets, redeploy the streaming functions, and run the Jordan / Ultimate Legends real-stream rehearsal.
```

## What Happens After Receipt

1. Set the five values in Firebase Functions secrets.
2. Redeploy the streaming functions.
3. Re-run the promoter control room stream issue flow.
4. Verify Mux stream key, ingest URL, playback ID, webhook, and replay path.
5. Capture proof for the Jordan lane.

## Secret Names

- MUX_TOKEN_ID
- MUX_TOKEN_SECRET
- MUX_SIGNING_KEY_ID
- MUX_SIGNING_PRIVATE_KEY
- MUX_WEBHOOK_SECRET

## Operator Note

Until these values exist, DFC should present the PPV lane as rehearsal-ready, not broken.
