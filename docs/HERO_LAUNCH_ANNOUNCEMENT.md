## 📊 BODY SECTION 4: The Technology

**How It Works**

1. **Orchestration Layer** — DFC ingests event markers from live broadcasts (timestamp, event type: knockout/submission/decision, fighter IDs)

2. **Auto-Clip Generation** — Computer vision + editorial algorithms select key frames, generate highlight clips with natural start/end points and audio sync

3. **Real-Time Feed Distribution** — Clips streamed via Firestore real-time listeners to creator dashboards and public viral feed

4. **Engagement Streaming** — View, like, share, and click events streamed to dashboard with 2–5 second latency

5. **PPV Attribution** — User clicks "Watch Full Fight on PPV" from clip modal → backend records conversion with idempotency key (prevents duplicates) + full audit trail

6. **Earnings Calculation** — Real-time aggregation: per-clip conversions × payout rate = immediate earnings visibility

7. **Payout Processing** — Biweekly batch processing with tax reporting + regulatory audit trail

**Technical Stack**:

- **Backend**: Firestore (real-time streaming), Cloud Functions (serverless processing)
- **Audit Trail**: Immutable append-only conversion logs (90-day retention, regulatory compliance)
- **Security**: Role-based access (only creator can read their own earnings), signed audit logs
