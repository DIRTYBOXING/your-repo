# Tier 6D Phase 3: E2E Validation & Integration Guide

## Overview

**Phase 3** is the comprehensive validation layer for the entire Tier 6D (Social Integration Loop). It provides:

1. **E2E Test Harness** (`e2e_test_harness.dart`) — 10-stage validation pipeline
2. **Orchestration Event Simulator** (`orchestration_event_simulator.dart`) — Simulates real fight events
3. **E2E Test Runner** (`e2e_test_runner.dart`) — 4 comprehensive test scenarios
4. **Debug Screen** (`e2e_debug_screen.dart`) — UI for manual testing

---

## Files Added

### 1. `lib/core/utils/e2e_test_harness.dart`

- **Purpose**: Full end-to-end validation pipeline
- **Class**: `E2ETestHarness`
- **10 Stages**:
  1. Setup (verify Firestore access)
  2. Knockdown Simulation (create orchestration event)
  3. Clip Generation (monitor auto-clip generation)
  4. Feed Update (verify social feed streaming)
  5. Engagement Simulation (record views, likes, shares)
  6. Trending Calculation (verify trending score updates)
  7. PPV Conversion (record conversion event)
  8. Creator Earnings (verify earnings calculation)
  9. Live Fight Banner (verify real-time sync)
  10. Cleanup (remove test data)

**Usage**:

```dart
final harness = E2ETestHarness();
final result = await harness.runFullValidation(
  eventId: 'event_test_001',
  sessionId: 'session_test_001',
);
print(result.summary);
```

### 2. `lib/core/utils/orchestration_event_simulator.dart`

- **Purpose**: Simulate real orchestration events for testing
- **Class**: `OrchestrationEventSimulator`
- **Methods**:
  - `simulateKnockdown()` — Create knockdown event
  - `simulateSubmission()` — Create submission event
  - `simulateRoundEnd()` — Create round end event
  - `simulateSequence()` — Simulate full round sequence
  - `getEventHistory()` — Retrieve all events for a fight

**Usage**:

```dart
final simulator = OrchestrationEventSimulator();
await simulator.simulateKnockdown(eventId, sessionId, fightId);
final events = await simulator.getEventHistory(eventId, sessionId, fightId);
```

### 3. `lib/core/utils/e2e_test_runner.dart`

- **Purpose**: High-level test scenario runner
- **Class**: `E2ETestRunner`
- **4 Scenarios**:
  1. **Scenario 1**: Full knockdown → clip → feed → engagement → PPV → earnings
  2. **Scenario 2**: Multiple events in sequence (complete round)
  3. **Scenario 3**: Engagement load test (100 users, 1000+ interactions)
  4. **Scenario 4**: Multiple creators with revenue distribution

**Usage**:

```dart
final runner = E2ETestRunner();
final result = await runner.runScenario1_FullKnockdownToEarnings();
print(result.summary);
```

### 4. `lib/core/screens/e2e_debug_screen.dart`

- **Purpose**: UI for manual testing (debug only)
- **Class**: `E2EDebugScreen`
- **Features**:
  - Status indicator (running/ready)
  - Scenario buttons with visual feedback
  - Real-time log panel
  - Color-coded success/error logs

**Usage**:

```dart
// Add to router
GoRoute(
  path: '/debug/e2e-tests',
  builder: (context, state) => const E2EDebugScreen(),
),

// Or navigate in code
context.push('/debug/e2e-tests');
```

### Step 3: Run Tests Programmatically

Use in any test or validation context:

```dart
void main() async {
  WidgetsFlutterBinding.ensureInitialized();
  await Firebase.initializeApp(options: DefaultFirebaseOptions.currentPlatform);

  final harness = E2ETestHarness();
  final result = await harness.runFullValidation(
    eventId: 'event_production_test_001',
    sessionId: 'session_production_test_001',
  );

  print(result.summary);
  exit(result.passed ? 0 : 1);
}
```

## Test Data Structure

All tests create Firestore documents under:

```
ppv_events/{eventId}/
  event_sessions/{sessionId}/
    fight_sessions/{fightId}/
      events/
        knockdown_001
        submission_001
        roundEnd_001
    social_clips/{clipId}
      (generated by AutoClipGeneratorService)
```

## Expected Flow

1. **Orchestration Event** → Created (knockdown, submission, round end)
2. **Auto Clip Generator** → Listens to events, generates clip
3. **Social Feed** → Streams new clip to feed
4. **Engagement Service** → Records views, likes, shares
5. **Trending Calculator** → Recalculates trending score
6. **PPV Conversion** → Records conversion event
7. **Creator Earnings** → Increments creator earnings
8. **Live Banner** → Updates real-time status
9. **Metrics** → Captured and logged

## Debugging Common Issues

### Clips Not Generating

- Verify `AutoClipGeneratorService` is initialized with correct `eventId` and `sessionId`
- Check Firestore rules allow read/write on social_clips collection
- Ensure orchestration events match expected format (type, round, timeInRound)

### Engagement Not Updating

- Verify `ClipEngagementService.recordView()` is called
- Check Firestore rules for clip_engagements collection
- Ensure clipId exists and is valid

### PPV Conversions Not Recording

- Verify `ClipAttributionService.recordClipConversion()` is called
- Check clip_conversions collection exists
- Ensure userId and clipId are valid

### Creator Earnings Not Calculating

- Verify `creator_earnings` collection has document for creatorId
- Check earnings calculation in `ClipAttributionService`
- Ensure PPV conversion was recorded first

## Metrics Captured

Each test captures:

- `totalDuration` — Full test duration
- `stageTimings` — Per-stage execution time
- `clipsGenerated` — Number of clips created
- `engagementRecorded` — Number of engagement events
- `conversionsRecorded` — Number of PPV conversions
- `earningsCalculated` — Creator earnings total
- `errorCount` — Number of errors encountered

## Firestore Data Validation Checklist

Before running full E2E tests, verify:

- [ ] `ppv_events/{eventId}` document exists
- [ ] `event_sessions/{sessionId}` document exists
- [ ] `fight_sessions/{fightId}` collection accessible
- [ ] `events` subcollection writable
- [ ] `social_clips` collection exists
- [ ] `clip_engagements` subcollection writable
- [ ] `clip_conversions` collection exists
- [ ] `creator_earnings` collection exists
- [ ] Firestore rules allow test reads/writes
- [ ] Firebase CLI authenticated correctly
