# DFC Good First Issues (Contributor Sprint List)

Use this list to onboard new contributors with low-risk, high-signal tasks.

## How to use

1. Pick one task with clear acceptance criteria.
2. Create an issue and tag it `good first issue`.
3. Link the issue in your PR using the contributor onboarding template.

## Sprint Tasks

### 1) Docs link integrity sweep
- Scope: `docs/launch/` and `docs/README.md`
- Acceptance: all links valid; no broken relative paths.

### 2) Outreach tracker helper notes
- Scope: add usage notes/examples to `DFC_SPONSOR_OUTREACH_TRACKER_TEMPLATE.csv`.
- Acceptance: contributors can use tracker in <10 minutes.

### 3) KPI snapshot sample row
- Scope: add one filled example to `KPI_SNAPSHOT_TEMPLATE.md`.
- Acceptance: template has clear reference example.

### 4) Partner onboarding checklist polish
- Scope: refine checklist wording for clarity and sequence.
- Acceptance: no ambiguity on required assets and timeline.

### 5) Contributor quickstart command audit
- Scope: verify command correctness and platform notes.
- Acceptance: quickstart works from clean clone.

### 6) README sponsor/grant section consistency
- Scope: align naming across README/docs for launch assets.
- Acceptance: consistent titles and link labels.

### 7) Add mission-safe language lint pass
- Scope: replace vague language with measurable outcomes in launch docs.
- Acceptance: docs read clearly for sponsors and grant reviewers.

### 8) Outreach email token validation
- Scope: ensure all outreach templates use consistent token syntax.
- Acceptance: no mixed token formats.

### 9) Add one new community-impact metric to KPI template
- Scope: introduce optional social-impact metric field.
- Acceptance: template remains concise and practical.

### 10) Docs navigation mini-index
- Scope: add small index block to major launch docs for quick navigation.
- Acceptance: faster reader scanning, no broken links.

## Definition of done

- Issue linked
- Focused PR
- Validation notes included
- Reviewer can verify in one pass
