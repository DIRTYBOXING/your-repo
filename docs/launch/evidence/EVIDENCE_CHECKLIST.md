# Grant Evidence Checklist

Use this checklist for each grant submission package.

## Narrative and positioning

- [ ] Problem statement is specific, measurable, and aligned to grant theme
- [ ] DFC solution summary is concise and technically credible
- [ ] Clear beneficiary definition (fighters, creators, youth, community)

## Execution proof

- [ ] Public roadmap updated
- [ ] Recent release cadence documented
- [ ] Operational runbooks and governance docs linked

## Technical credibility

- [ ] Architecture references included
- [ ] CI/quality gate evidence captured
- [ ] Reliability/security controls referenced

## Commercial/impact proof

- [ ] Sponsor deck and matrix attached
- [ ] Case study templates populated or linked
- [ ] KPI snapshot attached with targets vs actuals

## Submission quality control

- [ ] Owner and reviewer sign-off recorded
- [ ] Deadlines verified at T-48h and T-24h
- [ ] Final package version tagged and archived
