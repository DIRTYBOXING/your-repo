How to use

High-level architecture: devices/apps → ingestion/adapters → DFC brain → modules → experiences (charts, bots, camps).

2) Data Flows — ingestion → normalization → storage → AI → experiences

Shows the "truth pipeline" (what's collected, where it goes, and what triggers charts, bots, and training changes).

3) PPV Gates — "On and checked" for rapid builds

Pre-production → production readiness: what must be checked before launch. Use this as the definition of done.

4) Future Tech — chips, edge AI, new sensors (no rewrites)

Keep "godmode" sustainable: capability registry + model swap strategy + OTA updates, so new tech is plug-in work.
